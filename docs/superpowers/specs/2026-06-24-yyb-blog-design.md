# yyb 的 Blog 设计规格

## 目标

将当前 GitHub Pages 仓库初始化为 yyb 的个人 Blog，用来记录现场经验、材料观察、项目复盘和个人观点。

站点应像一个轻量的图文个人专栏，而不是企业产品官网。它需要突出作者自己拍摄的图片，并支持以后通过浏览器后台发布文章和上传图片。

## 视觉方向

采用“极简图文式”的个人专栏风格：

- 作者拍摄的图片应占据重要视觉位置。
- 文字应克制、清晰，重点放在经验、观察和观点上。
- 首页第一屏应通过图片和文章标题直接传达主题。
- 站点应避免营销型落地页的感觉。

## 技术方案

使用 Astro、Decap CMS 和 GitHub Pages。

Astro 负责生成静态网站。Decap CMS 提供 `/admin` 后台，让作者可以在浏览器里创建、编辑文章并上传图片。文章最终仍以 Markdown 文件保存在仓库中，上传图片保存在公开资源目录中。

## 站点结构

- 首页：包含图文优先的首屏、最新文章和简短个人介绍。
- Blog 列表页：展示全部文章。
- 文章详情页：展示封面图、元信息、标签和正文。
- 关于页面：介绍作者背景和写作目的。
- 后台页面：路径为 `/admin`，只给作者使用，不在公共导航中展示。

## 访问说明

普通访客访问 `/`、`/blog/` 和文章详情页时不需要登录。只有作者进入 `/admin` 管理内容时，才需要 GitHub 登录和仓库权限。
